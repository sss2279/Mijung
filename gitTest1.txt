버전 관리 시스템의 유형
# 어디에서 관리하는 지에 따라서 유형이 달라짐
- 로컬형
	- 사용자의 컴퓨터에서 관리하는 방식
- 중앙집중형
	- 중앙관리 서버에서 관리를 하는 방식
	- 중앙관리 서버에서 관리하기에 모두 사용가능
	- 중앙관리 서버가 다운됐을 경우 복구가 불가능하다는 단점이 존재
- 분산형
	- 서로 관리가 가능하다는 장점이 존재
	- 배워야 할 게 많다.

Git - 분산형 시스템에 속함

스테이지 - 업데이트를 곧 할 파일들을 확인하는 장소

언어 정리

커밋 - 파일의 변화 단위
초록색 + = 추가된 내용
빨간색 - = 없어진 내용


깃허브
클론 - 서버에 있는 데이터를 컴퓨터로 옮기는 작업
